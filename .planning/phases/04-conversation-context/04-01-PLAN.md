  <files>services/context.py, tests/test_context.py</files>
  <behavior>
    - resolve_uids("Hello <@U123> and <@U456>", client) returns "Hello Robert Li and Jane Doe" when those UIDs map to those names
    - resolve_uids with unknown UID keeps raw tag: "Hello <@UBAD>" stays as-is when API fails
    - resolve_uids with no UIDs returns string unchanged
    - resolve_uids caches: second call with same UID does NOT call users_info again
    - fetch_thread_history returns list of {role, content, type} dicts, bot messages as "assistant", others as "user"
    - fetch_thread_history excludes the current message (the one triggering the handler)
    - fetch_thread_history truncates messages longer than MSG_TRUNCATE_LENGTH
    - fetch_thread_history returns empty list when API call fails
    - fetch_channel_history returns list of {role, content, type} dicts for recent channel messages
    - fetch_channel_history returns at most HISTORY_DEPTH messages
    - fetch_channel_history returns empty list when API call fails
    - HISTORY_DEPTH defaults to 10, overridden by env var
    - MSG_TRUNCATE_LENGTH defaults to 500, overridden by env var
  </behavior>
  <action>
Create `services/context.py` with:

1. **Config constants** at module level:
   - `HISTORY_DEPTH = int(os.environ.get("HISTORY_DEPTH", "10"))`
   - `MSG_TRUNCATE_LENGTH = int(os.environ.get("MSG_TRUNCATE_LENGTH", "500"))`

2. **UID cache**: Module-level `_uid_cache: dict[str, str] = {}` (persists for bot lifetime, same pattern as `greeted_users` in shared.py).

3. **`resolve_uids(text: str, client) -> str`**:
   - Use `re.findall(r"<@([A-Z0-9]+)>", text)` to extract UIDs
   - For each UID not in `_uid_cache`: call `client.users_info(user=uid)` and cache `result["user"]["profile"]["display_name"]` or fall back to `result["user"]["real_name"]`. If display_name is empty string, use real_name.
   - On any exception for a UID lookup, skip it (leave raw tag)
   - Replace each `<@UID>` with the resolved name via `text.replace(f"<@{uid}>", name)`
   - Return modified text

4. **`_truncate(text: str) -> str`**: If `len(text) > MSG_TRUNCATE_LENGTH`, return `text[:MSG_TRUNCATE_LENGTH] + "..."`, else return text.

5. **`_build_message(text: str, bot_user_id: str, sender_id: str) -> dict`**:
   - Returns `{"type": "message", "role": "assistant" if sender_id == bot_user_id else "user", "content": _truncate(text)}`

6. **`fetch_thread_history(client, channel: str, thread_ts: str, current_ts: str, bot_user_id: str) -> list[dict]`**:
   - Call `client.conversations_replies(channel=channel, ts=thread_ts, limit=HISTORY_DEPTH + 1)` (extra 1 for the current message)
   - Filter out the message where `msg["ts"] == current_ts`
   - Take up to `HISTORY_DEPTH` remaining messages
   - Resolve UIDs in each message text via `resolve_uids()`
   - Build structured messages via `_build_message()` for each
   - Return list of dicts. On any exception, return `[]`

7. **`fetch_channel_history(client, channel: str, bot_user_id: str) -> list[dict]`**:
   - Call `client.conversations_history(channel=channel, limit=HISTORY_DEPTH)`
   - Resolve UIDs in each message text
   - Build structured messages via `_build_message()`
   - Return list in chronological order (reverse the API response which is newest-first). On any exception, return `[]`

Then create `tests/test_context.py` with tests for all behaviors listed above. Use `unittest.mock.MagicMock` for the Slack client, patch `os.environ` for config tests. Test the UID cache by calling resolve_uids twice and asserting users_info was called once.
